## §5 Departures from iter-3 / iter-4

- **FRAPPE entropy coding (`frp`, `seab`, `seaft`).** Iter-6 uses `compressors.frappe.entropy_coding` (per-scale `(C·H, W)` reshape, length-prefixed JPEG-LS streams) — the canonical pack also used by the FRAPPE encode-complexity harness. Iter-3 / iter-4 used per-channel JPEG-LS with no scale reshape, paying header overhead per plane. Iter-6 transmit bpp is therefore lower than iter-3 / iter-4 at matched n_ch; iter-6 numbers supersede those bpp values for the paper.
- **Distortion full-val.** Iter-3 reported PSNR / SSIM / LPIPS / DISTS on a 256-image subset of each task's validation split. Iter-6 reports the same metrics on the *full* validation split (50k cls, 2k seg). Throughput remains 256-image-subset.
- **Pipeline coverage.** Iter-6 adds three vanilla baselines absent from iter-3 / iter-4: `jp2` (JPEG 2000), `webp`, `wal` (WaLLoC RGB_16x).
- **`seaft` cls checkpoints.** Iter-6 uses the iter-5 cls fine-tunes (strictly dominate iter-1 C6 cls); iter-4 used iter-1 cls fine-tunes. Iter-6 supersedes for cls. Seg `seaft` continues to use iter-1.
- **`avifx` (libavif speed=10).** Carried over from iter-4's S5fast measurements unchanged; the iter-4 numbers map onto the iter-6 cells 1:1.
